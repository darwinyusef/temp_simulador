1 · Descripción general

El laboratorio simula la configuración de Router-on-a-Stick en la red de la Clínica San Rafael. El estudiante construye la topología arrastrando dispositivos a un canvas libre, conecta los elementos, asigna VLANs y configura las sub-interfaces del router mediante una terminal Cisco IOS interactiva para habilitar el routing entre tres VLANs hospitalarias.

Escenario

El departamento de TI de la Clínica San Rafael necesita que el personal de Administración (VLAN 10), Médicos (VLAN 20) y Enfermería (VLAN 30) puedan comunicarse entre sí, pero manteniéndose segmentados. Solo hay un enlace físico disponible entre el switch y el router.

Tecnologías simuladas

  • Cisco IOS 15.1 (CLI interactivo)
  • IEEE 802.1Q trunk encapsulation
  • Sub-interfaces Cisco (fa0/0.N)
  • Inter-VLAN routing a través de un único enlace
  • Ping test de verificación entre VLANs
🖱️
Interfaz drag-and-drop: A diferencia de laboratorios con topología fija, aquí el estudiante construye la red desde cero arrastrando dispositivos del panel lateral al canvas. El lab valida los tipos de nodos y conexiones requeridas, no posiciones exactas.

2 · Objetivos de aprendizaje

Al finalizar el laboratorio, el estudiante será capaz de:

Conceptuales

  • Explicar qué es y por qué se usa Router-on-a-Stick
  • Describir la estructura de una trama con etiqueta 802.1Q
  • Distinguir entre puerto de acceso y puerto trunk
  • Explicar cómo el router usa sub-interfaces para routing inter-VLAN
  • Identificar las ventajas y limitaciones de RoaS vs switches L3

Procedimentales

  • Construir una topología LAN desde cero (drag-and-drop)
  • Configurar un trunk 802.1Q entre switch y router
  • Crear sub-interfaces con encapsulation dot1Q
  • Asignar IPs gateway a cada sub-interfaz
  • Verificar conectividad inter-VLAN con ping test
🎯
Conexión curricular: Este lab construye sobre Lab 1 (VLANs conceptuales en un switch) añadiendo la capa de routing. Es prerrequisito directo de Lab 3 (Firewall & ACLs) y Lab 6 (Switch L2 avanzado), y establece la base para entender la segmentación de red en entornos con un único router de borde.

5 · Guía paso a paso

1
Topología Base — Construir desde cero
Drag-and-drop 15 pts ~5 min

Qué hace el estudiante

Arrastra desde el panel lateral los dispositivos necesarios: 1 Router, 1 Switch L2 y al menos 3 dispositivos finales (PCs, laptops o servidores). Los nodos se posicionan libremente en el canvas en blanco.

Requisito de unlock

  • Mínimo 1 nodo de tipo router en el canvas
  • Mínimo 1 nodo de tipo switch en el canvas
  • Mínimo 3 dispositivos finales (PC, laptop o servidor)

Puntos clave para reforzar

  • El router necesita un solo puerto físico — la magia de RoaS está en la configuración lógica (sub-interfaces)
  • Preguntar: ¿por qué no necesitamos un router por VLAN? → Las sub-interfaces son interfaces lógicas sobre el mismo puerto físico
  • El switch no necesita configuración especial en este lab (lo verán en Lab 6), aquí el foco es el router
⚠️
Error frecuente: El estudiante arrastra solo 2 PCs. El botón "Verificar Paso" mostrará el error "Necesitas al menos 3 PCs/dispositivos finales". Recordar que cada VLAN necesita al menos un host representativo.
2
Conexiones — Trunk 802.1Q
Canvas 20 pts ~8 min

Qué hace el estudiante

Usa la herramienta 🔗 Conectar para unir los PCs al switch (enlaces de acceso). Luego activa la herramienta 🔀 Trunk 802.1Q para crear el enlace trunk entre el switch y el router. El enlace trunk se visualiza distinto (línea doble o etiqueta).

Requisito de unlock

  • Todos los dispositivos finales deben tener al menos un enlace
  • Debe existir exactamente un enlace trunk entre un switch y un router

Diagrama objetivo

PC-Admin ──── [Switch L2] ════════ [Router]
PC-Medico ───┘   access          trunk 802.1Q
PC-Enfer ───┘   (sin tag)       (etiqueta VLAN en cada trama)
💡
Pregunta de discusión: ¿Por qué los puertos de acceso no llevan etiqueta 802.1Q? → Los dispositivos finales (PCs, impresoras) no entienden el encabezado 802.1Q. El switch añade y quita la etiqueta de forma transparente.
⚠️
Error frecuente: El estudiante conecta el switch al router con un enlace normal (no trunk). El simulador mostrará "Crea un enlace Trunk 802.1Q entre el Switch y el Router". Debe usar la herramienta de Trunk explícitamente.
3
Asignación de VLANs — Segmentación L2
Drag badges 15 pts ~5 min

Qué hace el estudiante

Arrastra las etiquetas de VLAN del panel sobre los dispositivos finales: VLAN 10 (azul) al PC de Administración, VLAN 20 (verde) al PC de Médicos, VLAN 30 (ámbar) al PC de Enfermería. Cada nodo muestra una insignia con el número de VLAN.

Requisito de unlock

  • Las 3 VLANs (10, 20 y 30) deben estar asignadas a al menos un dispositivo cada una

Puntos clave

  • Un dispositivo final solo puede tener una VLAN — los puertos de acceso pertenecen a una única VLAN
  • El switch conoce la VLAN de cada puerto; el router aún no — eso se configura en el paso 4
  • Las VLANs son segmentación lógica: dos PCs en VLANs distintas no pueden comunicarse directamente aunque estén en el mismo switch físico
🎨
Visualización: Cuando se asigna una VLAN, la insignia del dispositivo cambia al color de esa VLAN. Usar este momento para reforzar que el color representa el dominio de broadcast al que pertenece el host.
4
Sub-interfaces del Router — CLI Cisco IOS
CLI IOS 30 pts ~20 min

Secuencia correcta de comandos

-- Entrar al modo de configuración --
Router> enable
Router# configure terminal

-- Sub-interfaz para VLAN 10 (Administración) --
Router(config)# interface fa0/0.10
Router(config-subif)# encapsulation dot1Q 10
Router(config-subif)# ip address 10.10.10.1 255.255.255.0
Router(config-subif)# no shutdown
Router(config-subif)# exit

-- Sub-interfaz para VLAN 20 (Médicos) --
Router(config)# interface fa0/0.20
Router(config-subif)# encapsulation dot1Q 20
Router(config-subif)# ip address 10.10.20.1 255.255.255.0
Router(config-subif)# no shutdown
Router(config-subif)# exit

-- Sub-interfaz para VLAN 30 (Enfermería) --
Router(config)# interface fa0/0.30
Router(config-subif)# encapsulation dot1Q 30
Router(config-subif)# ip address 10.10.30.1 255.255.255.0
Router(config-subif)# no shutdown
Router(config-subif)# exit

-- Verificación desde conf mode (prefijo "do") --
Router(config)# do show ip interface brief
Interface          IP-Address       OK? Method Status
fa0/0.10           10.10.10.1       YES  manual up   up
fa0/0.20           10.10.20.1       YES  manual up   up
fa0/0.30           10.10.30.1       YES  manual up   up

-- O bien, salir a modo exec y verificar --
Router(config)# end
Router# show ip interface brief

Requisito de unlock

Las 3 sub-interfaces deben tener: encapsulation dot1Q configurado, dirección IP asignada y estado no shutdown (active). El modal CLI muestra el estado de cada sub-interfaz en tiempo real.

Conceptos a desarrollar en clase

ComandoFunciónAnalogía
interface fa0/0.10 Crea/selecciona sub-interfaz lógica sobre fa0/0 Como crear un carril exclusivo sobre la misma autopista
encapsulation dot1Q 10 Asocia la sub-interfaz a la VLAN 10; el router procesará tramas con tag VID=10 Asignar una etiqueta de identificación a cada carril
ip address 10.10.10.1 Define el gateway de la VLAN 10; los hosts usarán esta IP como default gateway Poner un portero en la entrada de cada carril
no shutdown Activa la sub-interfaz (por defecto están administrativamente down) Abrir la barrera del carril
do show ip interface brief Ejecuta un comando exec desde dentro del modo config/subif sin salir. Prefijo do estándar Cisco IOS. Asomarse por una ventana sin salir de la sala
❌
Error crítico frecuente: El estudiante configura ip address antes de encapsulation dot1Q. El CLI mostrará "Configure encapsulation primero.". Explicar que el router necesita saber qué VLAN procesar antes de asignar una IP a ese segmento.
5
Verificación — Ping Test Inter-VLAN
Simulación 20 pts ~5 min

Qué hace el estudiante

Abre el modal 🏓 Ping Test (botón en la barra de herramientas) y hace clic en Ejecutar Ping Test. El simulador verifica que las 3 sub-interfaces estén configuradas y activas, y que exista el trunk. Muestra el resultado de conectividad entre cada par de VLANs.

Resultado esperado

Verificando routing inter-VLAN...

VLAN10 → VLAN20: REACH  (via fa0/0.20, gw 10.10.20.1)
VLAN10 → VLAN30: REACH  (via fa0/0.30, gw 10.10.30.1)
VLAN20 → VLAN30: REACH  (via fa0/0.30, gw 10.10.30.1)

✓ Routing inter-VLAN: OK — Las 3 VLANs se comunican a través del router

Pregunta de verificación

❓

¿Por qué un ping de VLAN 10 a VLAN 20 pasa dos veces por el enlace trunk? → La trama va del host al switch (sin tag), el switch añade tag VLAN 10 y la envía al router; el router la procesa, crea una nueva trama con tag VLAN 20 y la devuelve al switch por el mismo enlace trunk. El switch quita el tag y la entrega al destino.

⚠️
Limitación de RoaS para discutir: Todo el tráfico inter-VLAN pasa por el mismo enlace físico. En redes con alto tráfico esto crea un cuello de botella. La solución es un switch de capa 3 con routing integrado — tema de Lab 6.

10 · Glosario técnico

Router-on-a-Stick (RoaS)
Técnica de inter-VLAN routing que usa un único enlace físico trunk entre un switch y un router, con sub-interfaces lógicas por VLAN en el router.
Sub-interfaz
Interfaz lógica creada sobre una interfaz física. Permite que un puerto físico gestione múltiples VLANs de forma independiente.
Encapsulation dot1Q
Comando Cisco que asocia una sub-interfaz a un VID específico. El router procesará todas las tramas etiquetadas con ese VID en esa sub-interfaz.
Trunk Port
Puerto que transporta tráfico de múltiples VLANs con etiquetas 802.1Q. Conecta switches entre sí o un switch con un router.
Access Port
Puerto que pertenece a una única VLAN. El switch añade/quita la etiqueta 802.1Q de forma transparente para el dispositivo final.
Inter-VLAN Routing
Proceso de reenviar tráfico IP entre distintas VLANs (distintos dominios de broadcast L2) a través de un router o switch L3.
